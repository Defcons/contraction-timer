# CODE-MAP — baby (apps monorepo)

_Last verified: 2026-07-19_

A family of small baby apps, one folder per app, each a static page on GitHub Pages + its own Cloudflare Worker for cross-device sync. Repo layout:

- **`index.html`** (root) — tiny landing page linking the apps; no logic, reuses either app's saved theme pref.
- **`contraction-tracker/`** — contraction timer (`index.html` + `worker/`).
- **`baby-tracker/`** — feeds/diapers/sleep tracker (`index.html` + `worker/`).
- **`pelvic-trainer/`** — postpartum pelvic floor exercise trainer (`index.html` + `worker/`).

Apps cross-link via small nav icons in the header (⏱️ / 🍼) but are fully independent — separate localStorage keys, separate sync rooms, separate Workers/KV namespaces. Each `worker/` dir contains `index.js` + its own `wrangler.toml`; deploy from inside that dir with `npx wrangler deploy` (needs `wrangler login`, account davidsen908).

History note: the repo began as `contraction-timer` with the timer at the root; it was renamed/restructured 2026-07-19. GitHub Pages URLs moved (`/contraction-timer/` → `/baby/contraction-tracker/`) but localStorage survived since it's keyed by origin, so existing devices kept their sync room + log without re-opening a share link.

## Contraction timer (`contraction-tracker/`)

- **`contraction-tracker/index.html`** — the whole app (vanilla JS, no build). Key symbols:
  - `pull` / `pushState` / `applyRemote` / `touch` — sync protocol: full-state last-write-wins, `revision` = `Date.now()` of last local mutation; poll every `POLL_MS` (4s) + on visibility/online.
  - `updateMainUI` — single place button/timer reflect `active`; called for both local taps and remote adoption. `tick` interval guard lives here.
  - Sync room comes from `#r=<id>` URL fragment (kept out of this public repo on purpose), then persisted in `localStorage[ROOM_KEY]`. No fragment ever seen → local-only mode.
  - `localStorage` v2 schema `{revision, contractions, active}`; one-shot migration from v1 in `load()`.
  - Theme: `data-theme` on `<html>`, CSS vars per theme, per-device pref in `localStorage['ctTheme']` (NOT synced). Inline head script applies it pre-paint.
  - Avg window: `#avgWindowSel` picks the span for the two avg stats — a contraction count (`5`/`8`/`10`) or a time span (`15m`/`30m`/`60m`/`120m`); per-device pref in `localStorage['ctAvgWindow']` (NOT synced). `windowSetForGaps`/`windowSetForDuration` resolve it against `contractions`; a 15s interval re-renders while a time-based window is active so contractions age out without a new tap.
  - `editRow`/`deleteRow` — per-row ✎/✕ via event delegation on `#logList` (`data-edit`/`data-del`); edit re-sorts by `start`, both call `touch()` to sync.
  - Deploy habit: snapshot live KV state to `backups/` (gitignored — room id is the credential) BEFORE pushing app changes.
- **`contraction-tracker/worker/index.js`** — Cloudflare Worker `contraction-sync` (KV binding `STATE`). GET/PUT `/state/:room`, CORS `*`, validates JSON + numeric `revision`.

Gotchas:
- GitHub Pages serves with `Cache-Control: max-age=600`: for up to ~10 min after a push, browsers (and the CDN) can serve the OLD page even though curl from another network sees the new one. Verified symptom: after a password-hash change, the correct new password gets "Wrong password" because the cached page still embeds the old hash — hard-refresh or wait, don't debug the app.
- Changing the family password moves every device to a different sync room in BOTH the baby tracker and the pelvic trainer (each derives its room from the password with its own salt), abandoning the old rooms' KV state — change it only when the logs are empty, or migrate the KV values first.
- KV is eventually consistent cross-colo (up to ~60s); same-household devices hit the same colo so sync is effectively instant. Don't "fix" apparent staleness when testing from different networks.
- Conflict model is wholesale LWW — two devices mutating in the same poll window can clobber one tap. Accepted: one person logs in practice.
- Worker PUTs can return CF edge error 1042 for ~1 min right after a fresh deploy — transient, retry.

## Baby activity tracker (`baby-tracker/`)

Local-first multi-device tracker. UNLIKE the contraction timer's wholesale LWW, sync is **entry-level merge** (see `sync.js`) so several caregivers logging concurrently never lose taps.

Installable PWA (`manifest.webmanifest` — has `id` + the page has `application-name` metas so Android recents shows "Baby Tracker" instead of "Web app"; a stale WebAPK may need uninstall/re-add to pick the name up — `sw.js`, generated icon PNGs) with web push alerts: shared alert rules live in the synced state (`alerts` key) and are evaluated **server-side** by the worker's cron so notifications arrive with the app closed; the push subscription itself is per-device.

- **`baby-tracker/sync.js`** — pure merge logic, no DOM (loaded by the page, eval'd by node tests). `normalizeState`/`mergeState`/`stateSig`. State: `{revision, entries, active, alerts, alertsRev, deleted, archivedBefore}`. Merge rules: entries union by `id` with higher `mt` (modified-ts) winning; deletes are tombstones in `deleted{id:ts}` (60-day TTL) that beat presence; `active` slots resolve by `mt` and are cleared when their stop-entry (`'s'/'n'+start`) exists; `alerts` is wholesale LWW by `alertsRev`; entries older than the server watermark `archivedBefore` are dropped (they live in archives). `stateSig` is a key-order-independent content signature used after each pull to decide adopt vs push — prevents push churn loops.
- **`baby-tracker/index.html`** — vanilla JS, no build. Key symbols:
  - Diaper and pump type icons are inline SVG `<symbol>`s (`#icoDiaper`/`#icoPump`, top of `<body>`; no emoji exists for either) referenced via `<use>`; `.svgico` is 1em so they scale with the surrounding font-size, and the JS-side strings live in `ICO_DIAPER`/`ICO_PUMP` (action buttons, timeline lane, filter chips, alerts row, log rows). Diaper *kind* markers stay 💧/💩 emojis.
  - Entry shape: `{id, type, start, end?, duration?, mt, by?, note?, ...type-specific fields}`; `type` is one of `sleep`/`nurse`/`bottle`/`diaper`/`solid`/`med`/`pump`/`health`. `nurse` has `side` (`L`/`R`/`both`); `bottle` has `amount`/`unit`/`milk`; `diaper` has `kind` (`wet`/`dirty`/`both`); `solid` has `food`; `med` has `name` (last name remembered per device in `btLastMed_v1`); `pump` has `amount`/`unit`/`side`; `health` has any of `temp`/`weight`/`length`. `stamp()` sets `mt` + `by` (per-device name, `btDeviceName_v1`, set in the 🔔 modal) on creation; every edit bumps `mt`.
  - Quick-log modals assume "today" for their time field; `applyTimeBack` treats a future-looking time as yesterday (night logging around midnight).
  - `editRow`/`deleteRow` address entries **by id**, not index — a 10s poll can reorder the array while a dialog is open. Deletes write a tombstone.
  - Log list renders newest-first with collapsible day sections (today always open; past days are one-line `▸` summary headers — feeds/diapers/sleep, or `n × icon` when filtered) plus a type filter chip row and an Expand all / Collapse all button in the Log heading; the day-collapse state is an in-memory Set (`expandedDays`) so sync re-renders don't reset it, while the filter persists per device (`localStorage['btLogFilter']`), and the `logLimit` row cap (50, "Show earlier" +100) counts only expanded entry rows; `renderHistory` shows 7-day totals (feeds / bottle ml / sleep / diapers); `csvBtn` exports full history to CSV, fetching server archives on demand.
  - `active = { sleep, nurse }` — two independent in-progress timers (mirrors the contraction timer's single `active`, just two of them since sleep and nursing can each be running/tracked concurrently). Tapping Sleep/Nurse while *idle* opens that type's modal (`openSleepModal`/`openNurseModal`): "Start timer" OR log an already-finished session via `makeWheel` scroll-snap wheel pickers (nurse: minutes 1–90; sleep: hours 0–16 + minutes step 5) — entry is backdated `end = now, start = end − duration`. Tapping while *running* stops directly, no modal. Bottle/Diaper/Solids are instantaneous events logged via a small modal (time defaults to now, editable). Logging nurse/bottle/diaper/solid/med while the sleep timer runs auto-stops the sleep at that event's time (`autoWake`) — events dated before the sleep began leave it running, and pump (mother's activity) / health (temp checks on a sleeping baby) never end it.
  - Login gate (`doLogin`/`sha256Hex`): full-screen `#loginOverlay` blocks the app until the shared password is entered; page embeds only `PASS_SHA256` (sha256 of the password — brute-forceable for a short password, accepted: it's bot deterrence, not security). The sync room id is *derived* from the password (`sha256('baby-room-v1:'+pw)` first 32 hex chars) so it never appears in the repo and every signed-in device lands in the same room automatically — no `#r=` link (that mechanism was removed from this app; the contraction timer still uses it). Persisted in `localStorage['babyAuthRoom_v1']`; presence of that key == signed in (no logout UI). `crypto.subtle` needs a secure context — https or localhost.
  - `pull`/`pushState`/`touch` — poll every `POLL_MS` (10s — one always-open tab ≈ 8.6k reads/day against the 100k/day free KV/Workers quotas; the interval skips polling while the tab is hidden, and visibilitychange pulls on return) + online pulls; `pull` merges via `mergeState` and pushes only when `stateSig` says the remote actually lacks something. Own localStorage key (`babyLog_v1`), own Worker (`SYNC_URL`).
  - Alerts UI (`RULES`/`paintAlertRules` + 🔔 modal): four rules — feed/diaper gap, awake-too-long, asleep-too-long — each `{on, min, crit}` (`crit`: `low`=silent/once, `normal`=sound/repeat 30 min, `high`="Nag" sound/repeat 10 min + sticky/renotify, `alarm`=strongest push every 5 min + rings locally), plus `quiet` hours `{on, from, to, tz}`. Rules are PER DEVICE: stored in `localStorage['btAlertPrefs_v1']` (`alertPrefs`, seeded from the old shared config on first run) and uploaded with the push subscription (`subscribePush` sends `{...sub, alerts}`); `savePrefs` also dual-writes into the legacy synced `alerts`/`bumpAlerts` field so an un-redeployed worker keeps working. Push side: `subscribePush`/`ensurePush`/`updatePushUI`, `VAPID_PUBLIC` embedded, per-device enabled flag `localStorage['btPushOn_v1']`; `ensurePush` re-POSTs the subscription on every load to survive push-service rotation. `updateSrvStatus` (modal open) POSTs `/status/:room` and shows last-cron-run + device-registered diagnostics. iOS: push only works installed to Home Screen (`isStandalone` check shows the hint).
  - Ringing alarm (`evalLocalAlarms`/`dueAlarmRules`/`stopAlarm` + `#alarmOverlay`): rules with crit `alarm` ring like a timer on any device where the app is open — WebAudio beep loop (unlocked on first pointerdown; silent until first user gesture) + vibration + full-screen stop overlay; evaluated every 15s and in `render()` so logging the activity (any device, via sync) silences it instantly; Stop snoozes 15 min (`ALARM_SNOOZE_MS`), condition clearing resets the snooze; respects per-device quiet hours (`localQuiet`).
  - `sw.js` — push + notificationclick handlers; same-origin GETs are network-first with cache fallback (offline works, deploys are never stale).
  - `renderTimeline` — activity strip with a 6h/12h/24h span toggle (per-device pref `localStorage['btTlSpan']`), one lane per type (`TL_TYPES`, 8 lanes): bars for sleep/nurse (incl. live `active` timers; nurse bars carry an L/R/B side letter), kind icons 💧/💩 on the diaper lane, dots for the rest. X-axis shows hour-aligned clock times (`tlTicks`; 6h/3h/2h label steps by span) with matching gridlines in the lanes, and dragging on the lanes places a scrubber line (`tlCursor` fraction + `paintTlCursor`) whose bubble shows the exact time / "ago" at that point — tap the bubble to dismiss; position survives re-renders. Re-rendered by `render()` + a 15s interval.
  - Layout: ≥760px (tablet / phone landscape) switches to a two-column grid (`.cols` — actions left, timeline/log/history right). The ⚡ top-bar button toggles a per-device "quick" mode (`localStorage['btUiMode']`): `body.quick` hides everything marked `.fullonly`, leaving only enlarged action buttons (4-across when wide) — works in both orientations.
  - `agoOr`/`lastOfType` — "X ago" sub-labels on all action buttons (refreshed in `updateStats`; running timers own their sub text via `updateActionUI`/`startTick`).
  - Edit modal has a date field (`combineDateTime`) so an entry can be moved to another day (e.g. mis-logged across midnight); sleep/nurse end times resolve on the start's day, an earlier clock time meaning midnight was crossed. It also shows/hides field groups per type (`editEndWrap`, `editSideWrap`, `editAmountWrap`, `editMilkWrap`, `editKindWrap`, `editFoodWrap`, `editMedWrap`, `editHealthWrap`); pump reuses amount+side wraps.
  - Theme: `localStorage['btTheme']` (separate from the contraction timer's `ctTheme`, per-device, NOT synced).
- **`baby-tracker/worker/`** — separate Cloudflare Worker `baby-tracker-sync`, its own KV namespace `BABY_STATE` (bound as `STATE` — binding name must stay `STATE` to match `env.STATE` in the worker code). Same GET/PUT `/state/:room` contract as the contraction timer's worker, plus push:
  - `webpush.js` — dependency-free Web Push sender (`sendPush`/`encryptPayload`/`vapidHeaders`): RFC 8291 aes128gcm + RFC 8292 VAPID via WebCrypto. Verified by an offline encrypt→decrypt round-trip test. VAPID private key = worker secret `VAPID_JWK` (JWK JSON, set via `wrangler secret put`, NEVER in the repo); the public key is embedded in `index.html`. Apple rejects JWTs with exp >24h — we use 12h.
  - `index.js` — `POST /subscribe|unsubscribe|test|status/:room`, `GET /archive/:room[/:month]`; `scheduled` (cron `*/5 * * * *` in wrangler.toml) runs `checkAlerts` + `dailyMaintenance`. `/subscribe` stores the device's alert prefs alongside the subscription (`{endpoint, keys, alerts?}`); `/status` returns `{lastCron, devices, subscribed}` for the in-app diagnostics line (`cron:last` heartbeat key, written at most every 30 min to spare the 1k/day KV write quota).
    - Subscriptions are stored as ONE blob per room (`subs:<room>` map keyed by endpoint hash) plus a `subsrooms` index, so the every-5-min cron does zero KV LIST operations — the free tier allows only 1k lists/day and the old per-key scheme burned ~600/day here (the cause of the July 2026 quota-warning emails). Legacy `sub:<room>:<hash>` keys migrate in lazily on first access and are deleted; identical re-subscribes skip the write.
    - `checkAlerts` (exported for tests): iterates `subsrooms`, evaluates `dueAlerts` PER DEVICE against that sub's own `alerts` config (fallback: the room state's legacy shared config for not-yet-updated clients; `dueAlerts` exported for tests; mirrors client stats logic, silent on empty logs, nursing/sleeping suppress feed/awake alerts; also always-on forgotten-timer guardrails `NURSE_TIMER_MS` 2h / `SLEEP_TIMER_MS` 14h), dedups via `alerted:<room>` map keyed `<deviceId>:<rule>` — fires on crossing, repeats per criticality (`REPEAT_MS`: low never / normal 30 min / high 10 min / alarm every cron run) while the condition holds, resets when it clears; low-crit pushes go out with `Urgency: normal`. `inQuietHours` (exported) is evaluated per device with that device's quiet config (tz via `Intl`; invalid tz fails open); suppressed alerts fire when the window ends.
    - `dailyMaintenance` (gated by `maint:last` date key, discovers rooms by listing `room:` keys): snapshots each room to `backup:<room>:<date>` (14 kept), then moves entries older than `HOT_DAYS` (35) into `archive:<room>:<YYYY-MM>` keys (union by id) and sets the `archivedBefore` watermark + a new revision on the trimmed hot state — this is what keeps the hot blob under the worker's 500KB PUT limit forever. Archived entries are read-only (history/CSV); clients drop them from hot copies via the watermark. 404/410 push responses prune the subscription.
- To share the baby tracker across devices, just sign in with the shared password on each device (room is derived from it — see login gate above).

## Pelvic trainer (`pelvic-trainer/`)

Guided pelvic floor (Kegel) trainer with synced session log + push reminders. Protocol follows NHS/NICE-style PFMT guidance (slow holds building to 10s ×10 + quick squeezes, 3 sessions/day). Installable PWA (manifest `id`, generated flower icon PNGs, `sw.js` network-first + push). Login gate uses the SAME family password as the baby tracker but a different room salt (`sha256('pelvic-room-v1:'+pw)` → `localStorage['pfAuthRoom_v1']`), so its data is a separate KV entry — and changing the shared password moves this app's room too. Main screen is compact (today card / start / plan / calendar); reminders and the technique guide live behind 🔔 and 📖 top-bar modals.

- **`pelvic-trainer/index.html`** — vanilla JS, no build. Key symbols:
  - Sync (`pull`/`pushState`/`mergeSessions`/`logSig`): sessions-only state `{revision, sessions:[{id,...}]}`, union-by-id merge (sessions are never deleted → no tombstones), poll every 60s (skipped while hidden) + visibility/online; `finishSession` pushes immediately. Pre-sync local entries get `id = 's'+ts` on load.
  - Session player (`buildSchedule`/`sessTick`/`startSession`): phase schedule = 3s prep → reps×(hold+rest) → 4s transition → quicks×(1s on/1s off); 100ms tick computes phase from elapsed wall time (drift-free, pausable via `pausedTotal`); animated circle scales per phase (transition duration matched to phase), WebAudio tones (unlocked on first pointerdown) + vibration cues per phase change, screen wake-lock held during a session (re-grabbed on visibilitychange).
  - Presets `PRESETS` (gentle 3s×8 / building 6s×10+5 / full 10s×10+10) + custom hold/rest/reps/quicks editor; config in `localStorage['pfCfg_v1']`.
  - Progress: sessions log `pfLog_v1` (`{ts, hold, rest, reps, quicks, preset}`); 3-dot today row, streak (consecutive days ≥1 session, today-not-yet-done doesn't break it), 5-week calendar heatmap (`paintProgress`), level-up hint after 5 days of ≥2 sessions on a non-full preset.
  - Reminders (🔔 modal): up to 3 per-device times (`pfRem_v1`) + enable toggle (`pfPushOn_v1`); the VAPID public key is FETCHED from the worker (`GET /vapid`, cached `pfVapid_v1`) so no key is hardcoded; `subscribePush` POSTs `{sub, times, tz, room}` (room lets the cron skip nudges once 3 sessions are logged that day); `ensurePush` re-POSTs on every load. Theme `pfTheme` (per-device).
  - Technique section: muscle identification, breathing/no-glutes rules, vaginal vs c-section start timing, the Knack, physio red flags + not-medical-advice note.
- **`pelvic-trainer/worker/`** — Cloudflare Worker `pelvic-reminders`, own KV namespace (binding `STATE`), own VAPID keypair (`genvapid.mjs` generates the JWK locally → `wrangler secret put VAPID_JWK`; public key served by `GET /vapid`). Routes: `GET/PUT /state/:room` (session-log sync, 200KB cap), `POST /subscribe|unsubscribe|test`, `GET /vapid`. All subscriptions live in ONE `subs` blob (keyed by endpoint hash; legacy `sub:<hash>` keys migrate in lazily) so the cron reads a single key instead of LIST-scanning; re-subscribes preserve the `sent` markers and skip the write when unchanged. Cron `*/5` runs `sendReminders` (exported for tests): per sub, compares the sub's local time (its stored `tz` via `Intl`) against its `times`, fires once per slot per local day (`sent` map, all updates aggregated into one blob write per run), skips the nudge when the sub's `room` already has 3 sessions that local day, prunes 404/410 endpoints. `webpush.js` is a copy of the baby tracker's sender. First-deploy setup: create KV namespace (`npx wrangler kv namespace create STATE`, paste id into wrangler.toml), set `VAPID_JWK` secret, deploy.
